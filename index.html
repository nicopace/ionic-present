“¡Quiero una aplicación Nativa!”

Las desventajas de las apps nativas

  • Se necesita experiencia en cada plataforma soportada
  • Código específico para cada plataforma
  • El desarrollo toma más tiempo

Cuantas más plataformas, más problemas.

¿Por qué aún seguimos programando para varias plataformas?

“¿Existe alguna alternativa?”
  • Aplicaciones híbridas: HTML5 que actúa como nativo
  • Phonegap que cambió de nombre a Cordova
  • Web encapsulado en una capa nativa
  • Acceso directo a APIs nativas
  • Entorno de desarrollo web familiar
  • Se desarrolla un único código para todas las plataformas (web)

Parado sobre los hombros de AngularJS

  • Probado en grandes apps web
  • Extiende el vocabulario HTML
  • Los componentes UI utilizan directivas y servicios

Enfocado a lo nativo

  • Modelado en base a Frameworks Nativos
  • Construido para trabajar junto a Cordova

La performance es lo primero

  • Animaciones aceleradas por hardware
  • Mínima manipulación del DOM
  • Elimina el delay de 300ms del tap

Basado en CSS

  • Sistema visual cohesivo
  • Limpio y simple
  • Fácil de personalizar
  • CSS independiente (independiente del JavaScript de Ionic)
  • ionicframework.com/docs/components

  • El CSS se genera en base al preprocesador SASS
  • Permite darle a la app tu imagen personal rápidamente
  • CSS diseñado para ser fácilmente adaptado
  • Basado en variables, con valores por defecto
  • Más de 80 mixins

Hoja de acción

  • Servicio AngularJS
  • Se inyecta en los controladores

$ionicActionSheet.show({
  titleText: 'Action Sheet Example',
  buttons: [
    { text: 'Share' },
    { text: 'Move' },
  ],
  destructiveText: 'Delete',
  cancelText: 'Cancel',
  buttonClicked: function(index) {
    console.log('BUTTON CLICKED', index);
    return true;
  }
});
